## 5. ХОД НА УРОКА

### **НОВ МАТЕРИАЛ**

#### **Структура на файлова система:**

```
ОБЩА СТРУКТУРА:

┌─────────────────────────────────────────────┐
│            Boot Block                        │
│  (Boot sector, bootstrap код)               │
├─────────────────────────────────────────────┤
│           Superblock / MBR                   │
│  (Информация за файловата система)          │
├─────────────────────────────────────────────┤
│          Inode Table / MFT                   │
│  (Метаданни за файловете)                   │
├─────────────────────────────────────────────┤
│           Data Blocks                        │
│  (Реалните данни на файловете)              │
└─────────────────────────────────────────────┘
```

#### **ext4 (Linux):**

```
ext4 структура:
├── Superblock - информация за FS
│   ├── Размер на блок (1KB, 2KB, 4KB)
│   ├── Общо блокове и inodes
│   ├── Свободни блокове и inodes
│   └── Timestamps
├── Block Group Descriptor Table
├── Block Bitmap - кои блокове са заети
├── Inode Bitmap - кои inodes са заети
├── Inode Table - метаданни за файлове
└── Data Blocks - съдържание

Характеристики ext4:
- Максимален файл: 16 TB
- Максимален дял: 1 EB (exabyte)
- Journaling (журнал за възстановяване)
- Extents (ефективно съхранение)
- Backwards compatible с ext3/ext2
```

#### **NTFS (Windows):**

```
NTFS структура:
├── Boot Sector
├── Master File Table (MFT)
│   ├── $MFT - самата MFT таблица
│   ├── $MFTMirr - копие на MFT
│   ├── $LogFile - журнал
│   ├── $Volume - информация за тома
│   ├── $AttrDef - дефиниции на атрибути
│   ├── $Root - главна директория
│   ├── $Bitmap - bitmap на клъстери
│   └── Записи за всеки файл
└── Data Area

Характеристики NTFS:
- Максимален файл: 16 EB (теоретично)
- Journaling
- ACL (разширени права)
- Compression, Encryption (EFS)
- Hard links, Symbolic links
- Alternate Data Streams (ADS)
```

#### **FAT32:**

```
FAT32 структура:
├── Boot Sector
├── File Allocation Table (FAT)
│   └── Таблица с указатели към клъстери
├── Root Directory
└── Data Area

Ограничения FAT32:
- Максимален файл: 4 GB
- Максимален дял: 2 TB (теоретично 8 TB)
- Без journaling
- Без права за достъп
- Съвместим с почти всичко
```

#### **Сравнение:**

| Характеристика | ext4 | NTFS | FAT32 |
|----------------|------|------|-------|
| Max файл | 16 TB | 16 EB | 4 GB |
| Max дял | 1 EB | 256 TB | 2 TB |
| Journaling | Да | Да | Не |
| Права | Unix | ACL | Не |
| Compression | Не | Да | Не |
| Encryption | Не* | EFS | Не |
| OS Support | Linux | Win/Linux | Всички |

### **ПРАКТИЧЕСКА РАБОТА**

**Linux команди:**
```bash
# Информация за файлови системи
df -Th                    # Монтирани FS
lsblk -f                  # Блокови устройства
sudo fdisk -l             # Дялове

# Информация за ext4
sudo dumpe2fs /dev/sda1 | head -50
sudo tune2fs -l /dev/sda1

# Създаване на файлова система (ВНИМАНИЕ!)
# sudo mkfs.ext4 /dev/sdX
# sudo mkfs.ntfs /dev/sdX
# sudo mkfs.vfat -F 32 /dev/sdX

# Монтиране
sudo mount /dev/sdX /mnt/point
sudo umount /mnt/point
```

**Windows:**
```cmd
# Информация
fsutil fsinfo volumeinfo C:
chkdsk C: /I

# Disk Management: diskmgmt.msc
```
