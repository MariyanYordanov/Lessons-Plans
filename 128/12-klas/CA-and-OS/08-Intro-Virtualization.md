### **3. НОВ МАТЕРИАЛ (40 минути)**

#### **Част 1: Какво е виртуализация? (10 минути)**

**Дефиниция:**
```
ВИРТУАЛИЗАЦИЯ е технология, която позволява създаването на 
виртуални (симулирани) версии на хардуер, операционни системи, 
storage устройства или мрежови ресурси.
```

**Аналогия:**
```
Физически сървър = Къща с един апартамент
Виртуализиран сървър = Жилищна сграда с много апартаменти

Същият хардуер, но много повече "обитатели" (VM)
```

**Видове виртуализация:**

| Тип | Описание | Примери |
|-----|----------|---------|
| Сървърна | Множество VM на един физически сървър | VMware, Hyper-V |
| Десктоп | Виртуални работни станции | VDI, Citrix |
| Мрежова | Виртуални мрежови устройства | NSX, SDN |
| Storage | Абстракция на съхранение | vSAN, Storage Spaces |
| Приложна | Контейнери | Docker, Kubernetes |

**История на виртуализацията:**
```
Timeline:
1960s - IBM CP/CMS (mainframe virtualization)
1999 - VMware Workstation
2001 - VMware ESX Server
2003 - Xen open-source hypervisor
2008 - Microsoft Hyper-V
2010 - Docker контейнери
2014 - Kubernetes
2024 - Cloud-native и edge computing
```

#### **Част 2: Hypervisor - Сърцето на виртуализацията (12 минути)**

**Дефиниция:**
```
HYPERVISOR (Virtual Machine Monitor - VMM) е софтуер, фирмуер 
или хардуер, който създава и управлява виртуални машини.

Hypervisor контролира достъпа на гост OS до физическия хардуер.
```

**Type 1 Hypervisor (Bare-Metal):**
```
┌─────────────────────────────────────────────┐
│              Виртуални машини               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │  VM 1   │ │  VM 2   │ │  VM 3   │       │
│  │ Ubuntu  │ │ Windows │ │ CentOS  │       │
│  └─────────┘ └─────────┘ └─────────┘       │
├─────────────────────────────────────────────┤
│         TYPE 1 HYPERVISOR                   │
│    (VMware ESXi, Hyper-V, Xen, KVM)         │
├─────────────────────────────────────────────┤
│              ХАРДУЕР                         │
│    (CPU, RAM, Storage, Network)             │
└─────────────────────────────────────────────┘

Характеристики:
✓ Инсталира се директно върху хардуера
✓ По-висока производителност
✓ По-добра сигурност
✓ Използва се в production среди
✓ Изисква специализирано управление
```

**Type 2 Hypervisor (Hosted):**
```
┌─────────────────────────────────────────────┐
│              Виртуални машини               │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐       │
│  │  VM 1   │ │  VM 2   │ │  VM 3   │       │
│  │ Ubuntu  │ │ Windows │ │ CentOS  │       │
│  └─────────┘ └─────────┘ └─────────┘       │
├─────────────────────────────────────────────┤
│         TYPE 2 HYPERVISOR                   │
│  (VirtualBox, VMware Workstation, Parallels)│
├─────────────────────────────────────────────┤
│           HOST ОПЕРАЦИОННА СИСТЕМА          │
│       (Windows 11, macOS, Linux)            │
├─────────────────────────────────────────────┤
│              ХАРДУЕР                         │
│    (CPU, RAM, Storage, Network)             │
└─────────────────────────────────────────────┘

Характеристики:
✓ Инсталира се като приложение върху OS
✓ По-лесна инсталация и използване
✓ Подходящ за development и тестване
✓ По-ниска производителност
✓ Зависи от стабилността на host OS
```

**Сравнителна таблица:**

| Характеристика | Type 1 | Type 2 |
|----------------|--------|--------|
| Производителност | Висока | Средна |
| Сигурност | Висока | Средна |
| Лекота на използване | Сложно | Лесно |
| Цена | Скъпо (корпоративен) | Безплатно/достъпно |
| Употреба | Production | Development/Testing |
| Host OS | Няма | Необходим |
| Примери | ESXi, Hyper-V, KVM | VirtualBox, VMware Workstation |

#### **Част 4: Хардуерна поддръжка и изисквания (8 минути)**

**Хардуерна виртуализация:**
```
Intel VT-x (Virtualization Technology)
├── Въведена: 2005 (Pentium 4)
├── Функция: Хардуерна поддръжка за виртуализация
├── EPT (Extended Page Tables): По-бързо memory management
└── VT-d: I/O виртуализация (passthrough)

AMD-V (AMD Virtualization)
├── Въведена: 2006 (Athlon 64)
├── Функция: Еквивалент на Intel VT-x
├── RVI (Rapid Virtualization Indexing): Memory optimization
└── AMD-Vi: I/O виртуализация

Проверка в Windows (PowerShell):
Get-ComputerInfo | Select-Object HyperVisorPresent, 
    HyperVRequirementVMMonitorModeExtensions

Проверка в Linux:
egrep -c '(vmx|svm)' /proc/cpuinfo
# Резултат > 0 означава поддръжка
```

**Минимални изисквания за hypervisor:**

| Компонент | Type 2 (VirtualBox) | Type 1 (ESXi) |
|-----------|---------------------|---------------|
| CPU | 64-bit с VT-x/AMD-V | 64-bit, 2+ cores |
| RAM | 4 GB (host) | 8 GB минимум |
| Storage | 50 GB HDD | 32 GB (bootable) |
| Network | 1 NIC | 1+ Gigabit NICs |

**Ресурси за виртуална машина:**
```
Виртуална машина (VM):
├── vCPU: 1-128+ виртуални процесора
├── vRAM: 1 MB - 6 TB памет
├── vDisk: VMDK, VHD, QCOW2 формати
├── vNIC: Виртуални мрежови адаптери
└── vGPU: Споделена или dedicated GPU

Препоръчителна конфигурация за Ubuntu Server VM:
├── vCPU: 2
├── vRAM: 4 GB
├── vDisk: 40 GB (thin provisioned)
└── vNIC: 1 (NAT или Bridged)
```

## 8. ПРИЛОЖЕНИЯ

### ПРИЛОЖЕНИЕ 1: ТЕРМИНОЛОГИЧЕН РЕЧНИК

```
╔══════════════════════════════════════════════════════════════╗
║              ТЕРМИНОЛОГИЯ НА ВИРТУАЛИЗАЦИЯТА                 ║
╠══════════════════════════════════════════════════════════════╣

HYPERVISOR (Virtual Machine Monitor - VMM)
  Софтуер, който създава и управлява виртуални машини.
  
HOST (Хост)
  Физическият компютър или OS, на който работи hypervisor.
  
GUEST (Гост)
  Виртуалната машина и нейната операционна система.
  
VM (Virtual Machine - Виртуална машина)
  Софтуерна емулация на физически компютър.
  
vCPU (Virtual CPU)
  Виртуален процесор, представящ дял от физическия CPU.
  
vRAM (Virtual RAM)
  Оперативна памет, разпределена за виртуална машина.
  
vDisk (Virtual Disk)
  Файл, който представлява твърд диск за VM.
  
THIN PROVISIONING
  Техника за разпределяне на storage при нужда.
  
THICK PROVISIONING
  Предварително заделяне на пълния размер storage.
  
SNAPSHOT
  Моментна снимка на състоянието на VM.
  
CLONE
  Пълно копие на виртуална машина.
  
TEMPLATE
  Шаблон за бързо създаване на нови VM.
  
LIVE MIGRATION
  Преместване на работеща VM между хостове.
  
HIGH AVAILABILITY (HA)
  Автоматично възстановяване при отказ на хост.
  
OVERCOMMITMENT
  Разпределяне на повече виртуални ресурси от физическите.
```

### ПРИЛОЖЕНИЕ 4: КОМАНДИ ЗА ПРОВЕРКА НА ВИРТУАЛИЗАЦИЯ

```bash
# ═══════════════════════════════════════════════════════════
#    КОМАНДИ ЗА ПРОВЕРКА НА ХАРДУЕРНА ВИРТУАЛИЗАЦИЯ
# ═══════════════════════════════════════════════════════════

# WINDOWS (PowerShell като Administrator)
# ─────────────────────────────────────────

# Проверка за Hyper-V съвместимост
systeminfo

# Детайлна проверка
Get-ComputerInfo | Select-Object `
    HyperVisorPresent, `
    HyperVRequirementDataExecutionPreventionAvailable, `
    HyperVRequirementSecondLevelAddressTranslation, `
    HyperVRequirementVirtualizationFirmwareEnabled, `
    HyperVRequirementVMMonitorModeExtensions

# Проверка дали Hyper-V е активен
Get-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V

# ─────────────────────────────────────────
# LINUX (Terminal)
# ─────────────────────────────────────────

# Проверка за CPU флагове (vmx = Intel, svm = AMD)
grep -E 'vmx|svm' /proc/cpuinfo

# Брой ядра с виртуализация
egrep -c '(vmx|svm)' /proc/cpuinfo

# Детайлна CPU информация
lscpu | grep Virtualization

# Проверка дали KVM модулът е зареден
lsmod | grep kvm

# Инсталиране на cpu-checker
sudo apt install cpu-checker
kvm-ok

# ─────────────────────────────────────────
# BIOS/UEFI НАСТРОЙКИ
# ─────────────────────────────────────────

# Ако виртуализацията е изключена:
# 1. Рестартирайте и влезте в BIOS/UEFI (F2, F12, Del)
# 2. Намерете: Advanced → CPU Configuration
# 3. Включете: Intel VT-x / AMD-V / Virtualization Technology
# 4. Запазете и рестартирайте

# ═══════════════════════════════════════════════════════════
```
